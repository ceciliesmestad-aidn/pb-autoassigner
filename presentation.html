Where to start

Going back to first principles works here too

1 Explain the problem and the context you work in
2 What connectors and tools do you have available?
3 What is the goal?
4 Plan and align
5 Execute, test and iterate
Applied to this problem

The context

250 unassigned notes in Productboard, needs routing to the right PM by domain
Productboard has an API — I read the docs and connected to it
Could also pull context from Slack and Notion if needed
Goal: grab new notes, categorise by domain, assign to PM — with human review before it touches PB
The build

Aligned, then said go

Claude read back the plan — we went back and forth until it was right
I wanted a visual interface, so it set up both a frontend and a backend
When it was done, I tested it and we iterated on feedback
Per-PM scope docs in YAML tell the model what each team owns — fully editable
